# Ideal Claude Project Structure

This folder shows the **recommended layout** for a project that uses the
Dev Intelligence Suite. Copy this structure into your own repository root
and populate with your actual code.

---

## Folder Responsibilities

### `.claude/` — Never edit manually
This is the Claude Project configuration. Claude reads skill definitions from here.
Install the Dev Intelligence Suite by copying the `.claude/` folder into your repo root.

### `src/` — Your code
Your actual backend and frontend code. Claude reads this but never writes to it.
Point skill prompts at `src/backend` and `src/frontend`.

### `graph/` — Graph outputs
Created automatically when you run `repo-graph-architect`. The `repo-graph.json` file
is consumed by all other skills — keep it updated whenever the repo structure changes
significantly (new modules added, modules deleted, dependencies changed).

**Refresh cadence**: Re-run the graph architect at the start of each sprint or
whenever a module boundary changes.

### `docs/` — Generated documentation
All Word documents and reports go here. Organised by type:
- `docs/frd/` — one `.docx` per system or module
- `docs/specs/` — API performance reports, architecture reviews
- `docs/gap-reports/` — gap analysis reports (versioned by date)

**Tip**: Commit these to source control. They become your living documentation.

### `tests/` — Generated test suites
The Playwright test generator scaffolds this entire directory. After generation,
edit the generated tests to add your specific test data and environment URLs.

**Tip**: Add real `.env` credentials and run `npx playwright test` immediately
after generation to verify the scaffold.

### `config/` — Suite configuration
Read-only reference files. Edit `workflow-config.json` to customise trigger phrases
or output paths for your team's conventions.

## File Naming Conventions

| File | Convention | Example |
|---|---|---|
| FRD documents | `[system-name]-frd-v[X.Y].docx` | `orders-frd-v1.0.docx` |
| Gap reports | `gap-analysis-[YYYY-MM-DD].md` | `gap-analysis-2024-01-15.md` |
| UI Page Objects | `[feature]-page.ts` | `order-list-page.ts` |
| UI Specs | `[feature].spec.ts` | `orders.spec.ts` |
| API Specs | `[resource]-api.spec.ts` | `orders-api.spec.ts` |
| Graph files | `repo-graph.json` / `repo-graph.html` | (fixed names) |
