# Story Readiness Checklist

Before a story goes to Developer (Dev), verify:

- [ ] Story has a clear "As a... I want... So that..." description
- [ ] At least 2 acceptance criteria defined
- [ ] Repo target specified (Java / Angular / Both)
- [ ] Dependencies on other stories listed
- [ ] "Existing Code to Reference" section populated with real file paths
- [ ] Every task has a full file path within the monorepo
- [ ] Cross-repo stories have tasks split into Java + Angular sections
- [ ] Verification tasks present for any cross-boundary story
- [ ] Architecture doc referenced for API contracts
